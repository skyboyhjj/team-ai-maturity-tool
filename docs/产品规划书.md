# 团队AI成熟度自评工具 · 产品规划书 v1.1

> **"评测他人的工具，自己先成为被评测的样本。"**
>
> 本规划书将自评工具从「人工填表（阶段1）→ 机器自动采+人工补（阶段2→3）→ 季度自评自动化（阶段4）」的进化路径完整呈现。工具的每一阶段升级，对应它自己所评测的 AI 采用框架的同一阶段——这是自指涉的工程实践。

---

## 一、产品愿景

**让每一个团队都能像看健康手环一样，实时看见自己的AI成熟度。**

当前状态：一个HTML单页，人打开→手动填15题→点按钮出报告。

目标状态：一套自动化流水线，每周从工作流中自动采集客观指标，每季度轻量人工复核一次主观指标，趋势自动追踪，报告自动推送。

## 五、阶段3：智能体自主评测（📋 规划中）

### 5.1 目标

**智能体主动从数据源采集指标，自动生成诊断建议，仅在异常时通知人介入。人从"操作者"变成"审核者"。**

### 5.2 架构升级

```
阶段2                              阶段3

人触发 GitHub Actions               智能体定时自主触发
     │                                    │
     ▼                                    ▼
  采集脚本 ──→ 写入表格        Agent 主动采集 ──→ 写入表格
     │                                    │
     ▼                                    ▼
  人工补主观题                   Agent 自动发现异常
     │                                    │
     ▼                                    ▼
  手动合并                       Agent 生成诊断建议
                                        │
                                        ▼
                                  人只在异常时审核
```

### 5.3 新增能力

| 能力 | 说明 |
|:-----|:-----|
| **自主触发** | AI Agent 按周/月自动启动采集流水线，无需人触发 GitHub Actions |
| **异常自动诊断** | 当某项指标出现显著偏离（如AI代码占比突然下降20%），Agent 自动分析可能原因并推送告警 |
| **上下文感知** | Agent 能读取团队Wiki、企微讨论、项目Board，理解"为什么这个指标在变化" |
| **建议生成** | 根据异常模式，从行动清单中自动匹配对应建议（如"缘位轴连续下降→建议检查CI配置是否过期"） |
| **趋势预测** | 基于历史数据，预测下季度可能进入哪个阶段，提前给出准备建议 |
| **自动补主观题** | 通过分析企微群中的AI相关讨论频次、情感倾向，部分替代信任度和角色感的人工评测 |

## 六、阶段4：全自动闭环（🔮 远期）

### 6.1 目标

**评测完全嵌入工作流，零人工介入。指标从"事后采集"变为"事中感知"，评测工具退化为无形的基础设施。**

### 6.2 终极形态

| 维度 | 阶段4形态 |
|:-----|:--------|
| **数据源** | 指标不再"采集"——CI/CD、Git、IM 工具原生输出成熟度指标 |
| **评测触发** | 不再"定时跑"——每次代码提交、每次Agent调用，自动更新成熟度 |
| **人机关系** | 人不再"填表"也不再"审核"——只在飞书仪表盘上扫一眼趋势，有异常时AI已经处理完并附处理结果 |
| **演化能力** | 评测体系自我迭代——AI发现某个指标不再有区分度，自动建议调整问卷题目 |

### 6.3 与AI采用框架的自指涉

```
评测工具本身的进化 = 它所评测的框架的活样本

阶段1：人工填表     →  评测工具的 "辅助模式"
阶段2：自动采+人工补 →  评测工具的 "并行模式"
阶段3：Agent自主评测 →  评测工具的 "自主运行"
阶段4：全自动闭环   →  评测工具的 "AI原生"
```

这意味着：当这个工具的月度报告开始自动推送到企微群、人只需要扫一眼趋势时，**工具已经达到了它所评测的"阶段3→4"**——它不仅在评测团队的AI成熟度，它本身就是团队AI成熟度的活证据。
