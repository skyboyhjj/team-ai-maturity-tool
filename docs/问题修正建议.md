## 问题 1：`getAnswers()` 中 `if(val)` 的类型隐患

### 当前代码

```javascript
function getAnswers(){
  const answers = {};
  document.querySelectorAll('.q').forEach(q=>{
    const val = q.dataset.answer;
    if(val) answers[parseInt(q.dataset.q)] = parseInt(val);
  });
  return answers;
}
```

### 隐患分析

当前逻辑依赖一个 JavaScript 特性：**非空字符串在 `if()` 中为 truthy**。

| 场景 | `q.dataset.answer` | `if(val)` | 结果 |
|:-----|:-------------------|:---------|:-----|
| N/A 选项 | `"0"`（字符串） | truthy ✅ | 进入 answers（值=0） |
| 正常选项 | `"1"` ~ `"5"` | truthy ✅ | 进入 answers |
| 未答题 | `undefined` | falsy ✅ | 不进入 answers |

现在正确。但如果未来维护者**把 N/A 赋值改为数字 `0`**（如 `q.dataset.answer = 0`），`if(0)` 变成 falsy，N/A 题目会被跳过，导致 `totalAnswered < 15`，永远无法通过检查——工具彻底不可用。

### 建议修改

```javascript
function getAnswers(){
  const answers = {};
  document.querySelectorAll('.q').forEach(q=>{
    const val = q.dataset.answer;
    if(val !== undefined && val !== null) {
      answers[parseInt(q.dataset.q)] = parseInt(val);
    }
  });
  return answers;
}
```

`!== undefined && !== null` 同时覆盖了 N/A（值 0）和未答题（undefined），不依赖字符串 truthy 特性。

## 问题 2：`totalAnswered` 命名有歧义

### 当前代码

```javascript
function calculate(){
  const answers = getAnswers();
  const totalAnswered = Object.keys(answers).length;  // 含N/A
  if(totalAnswered < 15){
    alert('请先完成全部 15 道题（可选择"N/A"表示不确定）');
    return;
  }

  const validAnswers = {};
  let naCount = 0;
  for(const [q,val] of Object.entries(answers)){
    if(val > 0) validAnswers[q] = parseInt(val);
    else naCount++;
  }
  const validCount = Object.keys(validAnswers).length;  // 不含N/A
```

### 问题分析

函数内有两个概念相近的变量：

| 变量 | 含义 | 是否含 N/A |
|:-----|:-----|:---------:|
| `totalAnswered` | 已点击选项的题数 | ✅ 含 N/A |
| `validCount` | 给出了有效分数的题数 | ❌ 不含 N/A |

命名上 `Answered` 暗示"已作答"，而 N/A 确实是一种作答（选了"不确定"）。但后续 `validCount` 才是真正参与评分的题目数，两者不对比阅读容易混淆其语义。

### 建议修改

将 `totalAnswered` 改为更明确的名字：

```javascript
const allClicked = Object.keys(answers).length;
if(allClicked < 15){
  alert('请先完成全部 15 道题（可选择"N/A"表示不确定）');
  return;
}
```

或者始终用 `answeredCount` + `scoredCount` 配对的命名模式，让读者一目了然两者区别。

## 问题 3：`autoCalc()` 完全静默，用户填完后无任何反馈

### 当前代码

```javascript
function autoCalc(){
  const answers = getAnswers();
  if(Object.keys(answers).length < 15) return;
  // 静默计算，不弹窗，不显示浮动提示
  // 用户主动点击"生成诊断报告"时才展示结果
}
```

函数体是空的——只有注释，没有执行任何操作。

### 问题分析

这是有意为之的设计简化，但存在用户体验问题：

- 用户答完第 15 题后，页面**毫无变化**——没有提示、没有按钮高亮、没有任何"已完成"的信号
- 用户可能困惑："填完了吗？下一步怎么办？"
- 需要自己滚动到底部点击「生成诊断报告」，但这个按钮的位置不一定显眼

### 建议修改

在 `autoCalc()` 全部答完后给出一个最小化的视觉提示：

```javascript
function autoCalc(){
  const answers = getAnswers();
  if(Object.keys(answers).length < 15) return;

  // 全部答完 → 轻提示：引导点击底部按钮
  const actionBtn = document.querySelector('.btn.primary');
  if(actionBtn){
    actionBtn.style.animation = 'pulse 1.5s ease-in-out 3';
    actionBtn.textContent = '📊 全部完成，点击生成报告';
  }
}
```

配合 CSS 动画：

```css
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}
```

效果：用户填完最后一题的瞬间，底部按钮轻微脉动 3 次 + 文案从"生成诊断报告"变为"全部完成，点击生成报告"——不打扰用户，但给出了明确的"下一步"信号。
